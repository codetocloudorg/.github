## Bugs Found

### BUG-001: WSL "Access Denied" on Windows Server 2022 in Azure

**Severity:** Medium  
**Component:** Kodra-WSL  
**Environment:** Azure Windows Server VM

**Description:**  
When attempting to use WSL commands (`wsl --status`, `wsl --list --online`) on Windows Server 2022 in Azure, the command returns "Access is denied."

**Steps to Reproduce:**
1. Create Windows Server 2022 VM in Azure
2. Enable WSL feature via PowerShell: `Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux`
3. Enable Virtual Machine Platform: `Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform`
4. Restart VM
5. Run `wsl --install` or `wsl --status`

**Expected Result:**  
WSL should install and operate normally

**Actual Result:**  
"Access is denied" error

**Root Cause:**  
Azure VMs require nested virtualization for WSL2 to function. Standard VM sizes may not support this feature.

**Remediation:**
1. Use Azure VM sizes that support nested virtualization (Dv3, Ev3, Dv4, Ev4 series)
2. Enable nested virtualization on the VM
3. Or use Windows 11/10 Pro/Enterprise which has better WSL support
4. **Documentation Update Required:** Add prerequisites to Kodra-WSL cheatsheet
